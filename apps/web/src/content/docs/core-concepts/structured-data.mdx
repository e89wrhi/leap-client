# Structured Data (JSON) Extraction

One of the most powerful features of Leap AI SDK is the ability to generate specific data structures natively from C# types using `GenerateObjectAsync<T>`.

## GenerateObjectAsync

The `GenerateObjectAsync<T>` method automatically parses your C# Records or Classes into standard JSON Schemas and instructs the AI to output matching JSON. Leap then deserializes this JSON back into your strong type automatically.

```csharp
using Leap.AI.Core;
using Leap.AI.Providers.OpenAi;

// Define your highly nested model
public record CharacterSetup(string Name, string Class, int Level, List<string> Inventory);

var leap = LeapClient.Create().UseOpenAi("sk-...", "gpt-4o").Build();

// Generate it!
CharacterSetup character = await leap.GenerateObjectAsync<CharacterSetup>(
    "Generate a level 5 elven ranger for an RPG, with 3 signature items in their inventory."
);

Console.WriteLine($"Name: {character.Name}");
Console.WriteLine($"Class: {character.Class}");
Console.WriteLine($"Items: {string.Join(", ", character.Inventory)}");
```

## How it Works

1.  **Schema Generation**: Leap uses reflection to extract a JSON Schema from your C# type `<T>`.
2.  **Provider Integration**: This schema is passed to the AI provider (OpenAI, Anthropic, etc.) using their native structured output features (like JSON Mode or Tool Choice).
3.  **Automatic Mapping**: Once the AI returns the JSON string, Leap maps it back to your object, handling type conversions and nesting.

## Best Practices for Structured Data

-   **Descriptive Properties**: Use clear names for your properties. The AI uses these names to understand what data to put where.
-   **Model Selection**: Complex schemas require more capable models. Use `gpt-4o`, `claude-3-5-sonnet`, or `gemini-1.5-pro` for reliable JSON generation.
-   **Validation**: While Leap handles the mapping, it's always good practice to validate the data logic in your application after it's generated.
