# Chat & Streaming API

The Leap AI SDK provides both synchronous and streaming interfaces for chat completions over a middleware pipeline.

## Generating Text

`GenerateTextAsync` is the simplest way to get a response from an AI model. It handles the request-response cycle and returns the final string.

```csharp
var leap = LeapClient.Create()
    .UseOpenAi("sk-...", "gpt-4o")
    .Build();

var response = await leap.GenerateTextAsync("Hello! Write me a short story.");
Console.WriteLine(response);
```

### Working with Chat Messages

For multi-turn conversations, you can pass a list of `ChatMessage` objects, which include roles like `System`, `User`, and `Assistant`.

```csharp
var messages = new List<ChatMessage> {
    ChatMessage.System("You are an aggressive pirate."),
    ChatMessage.User("Give me directions to the store.")
};

var response = await leap.GenerateAsync(messages);
```

## Streaming Completions

Streaming allows you to receive tokens as they are generated, providing a more responsive user experience, especially for interactive applications.

```csharp
await foreach (var chunk in leap.StreamAsync("Write a 5 paragraph essay on deep learning."))
{
    Console.Write(chunk.Text);
}
```

## Middleware & Options

You can configure global behavior for your pipeline, such as logging and retries, during the build phase:

```csharp
var leap = LeapClient.Create()
    .UseOpenAi("sk-...", "gpt-4o")
    .UseLogging() // Auto-log AI pipeline events
    .UseRetry(maxRetries: 3) // Native exponential backoff
    .Build();
```

## Best Practices

1.  **Cancellation**: Always pass a `CancellationToken` to `StreamAsync` or `GenerateAsync` to allow users to stop long-running generations.
2.  **System Prompts**: Start your conversations with a `ChatMessage.System()` to provide high-level instructions to the model.
3.  **Error Handling**: Wrap your calls in try-catch blocks to handle potential network or API errors from the providers.
